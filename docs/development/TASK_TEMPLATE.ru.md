# Шаблон task bundle

## TASK.ru.md

### 1. Заголовок
Короткое и точное название задачи.

### 2. Контекст
Что уже есть в проекте и почему задача возникла сейчас.

### 3. Постановка задачи
Что именно должно измениться.

### 4. Scope
Что входит в задачу.

### 5. Non-goals
Что специально не входит в задачу.

### 6. Затрагиваемые области
Какие каталоги, подсистемы и документы будут затронуты.

### 7. Критерии приёмки
Проверяемые условия, по которым задача считается завершённой.

Для workflow/governance tasks дополнительно:
- какие instruction surfaces должны остаться согласованными;
- что считается reusable core и что остаётся project-specific overlay;
- какие machine-checkable guardrails должны быть прогнаны.

Для baseline-porting tasks дополнительно:
- из какого repo взят baseline;
- какие reusable surfaces копируются;
- какие overlay skills/agents заменяются или удаляются;
- какие repo-specific docs должны быть переписаны до первого product slice.

### 8. Риски
Что может пойти не так.

## PLAN.ru.md

### 1. Current phase
На каком этапе разработки находится задача.

### 2. Orchestration
Какой режим выбран: `solo`, `light` или `full`, и почему.

Если режим не `solo`, какие read-only reviews нужны до реализации.

### 3. Slices
Разбивка задачи на маленькие шаги.

Для каждого slice:
- цель;
- какие файлы/каталоги меняются;
- какие проверки нужно выполнить;
- какой артефакт результата должен появиться.

### 4. Rollback point
Где можно безопасно остановиться и откатиться без поломки структуры.

### 5. Final validation
Какие репозиторные проверки обязательны перед завершением.

Для workflow/governance tasks сюда входит `make lint-codex-governance`.
