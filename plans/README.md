# plans/

Здесь лежат task bundles для реальных задач.

Структура каталога:
- `plans/active/<slug>/` — активные bundles, по которым сейчас ведётся работа;
- `plans/archive/<year>/<slug>/` — завершённые bundles, оставленные как
  инженерная история и контекст решений.

Формат папки:
- `plans/active/<slug>/TASK.ru.md`
- `plans/active/<slug>/PLAN.ru.md`
- при необходимости `REVIEW.ru.md`, `NOTES.ru.md`, `DECISIONS.ru.md`

Задача считается нормально подготовленной только если сначала появился task bundle, а уже потом начались изменения кода или templates.

Правила hygiene:
- корень `plans/` не должен разрастаться произвольными slug-каталогами;
- после завершения задачи bundle переносится в `plans/archive/<year>/`, если он
  ещё нужен как история;
- мелкие одноразовые bundles можно удалять после merge, если они не несут
  долгоживущей архитектурной ценности;
- `plans/` хранится в git, но не должен попадать в release artifacts модуля.
