# Сделать `make verify` канонической repo-level проверкой

## Контекст

В репозитории уже используется `make verify` как основной repo-level validation
loop. При этом в `AGENTS.md` до сих пор фигурирует `task verify` как будто это
равноправная рекомендуемая команда, что противоречит ожидаемому рабочему циклу.

## Постановка задачи

Нужно закрепить `make verify` как каноническую команду проверки репозитория и
убрать из repo rules указание использовать `task verify` вместо неё.

## Scope

- поправить repo rules и guidance, где `task verify` фигурирует как рекомендуемая
  команда проверки;
- оставить `Taskfile` только как вспомогательный wrapper, если он не мешает
  policy.

## Non-goals

- не убирать весь `Taskfile` из репозитория;
- не перестраивать validation pipeline;
- не менять состав `make verify`.

## Затрагиваемые области

- `plans/canonicalize-make-verify/`
- `AGENTS.md`

## Критерии приёмки

- в repo rules `make verify` указан как canonical repo-level check;
- `task verify` больше не фигурирует как рекомендуемая обязательная команда;
- `make verify` проходит.

## Риски

- если убрать слишком много, можно случайно сломать локальные wrappers без нужды;
- если исправить только часть guidance, policy останется двусмысленной.
