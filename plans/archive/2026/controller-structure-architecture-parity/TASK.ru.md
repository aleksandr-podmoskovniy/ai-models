### 1. Заголовок

Актуализация `images/controller` structure map и архитектурный аудит

### 2. Контекст

`images/controller/STRUCTURE.ru.md` отстал от текущего runtime tree:
появились отдельный `ai-models-node-cache-runtime`, CSI node server,
node-cache runtime dataplane, in-process artifact cleanup и новые adapter/support
boundaries. Документ должен снова описывать живую архитектуру, а не старую
transition-картину.

### 3. Постановка задачи

Выровнять `images/controller/STRUCTURE.ru.md` с фактическим кодом и проверить,
что текущие boundaries соответствуют Deckhouse/virtualization-style module
patterns, hexagonal/SOLID discipline, LOC/complexity constraints and no
monolith drift.

### 4. Scope

- `images/controller/STRUCTURE.ru.md`;
- `docs/CONFIGURATION*.md` and OpenAPI descriptions touched only to keep
  node-cache size semantics consistent with code;
- `templates/_helpers.tpl` and helm render validation tests touched only to
  make the node-cache size guard and webhook fail-closed proof fail-fast;
- read-only architecture audit по `images/controller/**`;
- точечные code/docs cleanups только если аудит найдёт очевидный drift без
  отдельного product decision.

### 5. Non-goals

- Не превращать `STRUCTURE.ru.md` в полный autogenerated listing всех функций:
  это создаст docs-монолит, который невозможно поддерживать.
- Не менять публичный API, values schema или RBAC в этом slice.
- Не переписывать большие runtime paths без отдельного implementation slice.
- Не трогать уже существующие unrelated dirty изменения.

### 6. Затрагиваемые области

- Controller/runtime architecture docs.
- Потенциально small cleanup внутри `images/controller/internal/nodecache`,
  `dataplane/nodecachecsi`, `dataplane/nodecacheruntime` или controller
  boundaries, если найдены явные несоответствия.

### 7. Критерии приёмки

- `STRUCTURE.ru.md` перечисляет актуальные execution entrypoints и live package
  map.
- Документ объясняет ответственность новых boundaries:
  `nodecachecsi`, `dataplane/nodecacheruntime`, `cleanupstate`,
  `archiveio`; empty `publishplan` placeholder is removed instead of being
  documented as a fake boundary.
- Документ явно фиксирует, где должна жить file/function-level детализация:
  в package comments, tests and evidence docs, а не в giant architecture file.
- Архитектурный аудит фиксирует найденные risks/decisions в plan notes.
- Если есть низкорисковые реальные несоответствия, они исправлены одним узким
  patch.
- Проверки по затронутым пакетам и repo-level guards пройдены.

### 8. Риски

- Полный function inventory внутри `STRUCTURE.ru.md` быстро устареет и станет
  источником ложной архитектурной правды.
- В рабочей копии уже есть много staged/dirty изменений, поэтому refactor надо
  делать только в явно затронутой области и не откатывать чужие изменения.
