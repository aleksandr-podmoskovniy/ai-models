# Archive Stale Active Bundles

## 1. Контекст

`plans/active/` разросся и больше не отражает реально активную работу.
Завершённые design/implementation slices остались в active, из-за чего:

- текущий рабочий контекст размыт;
- сложно понять, какой bundle действительно живой;
- нарушаются правила hygiene из `plans/README.md`.

## 2. Постановка задачи

Провести hygiene-cleanup для `plans/active/`:

1. выделить завершённые и уже исторические bundles;
2. перенести их в `plans/archive/2026/`;
3. оставить в `plans/active/` только реально живые или неизвестные без
   безопасного сигнала для архивации bundles;
4. зафиксировать, какие каталоги были перенесены и почему.

## 3. Scope

- `plans/active/*`
- `plans/archive/2026/*`
- текущий bundle cleanup-задачи

## 4. Non-goals

- Не переписывать содержание исторических bundle-файлов.
- Не удалять bundles, если их статус нельзя безопасно определить.
- Не менять код, templates, API или runtime.

## 5. Критерии приёмки

- В `plans/active/` остаются только текущий cleanup bundle и bundles, которые
  нельзя безопасно считать завершёнными.
- Завершённые historical slices перенесены в `plans/archive/2026/`.
- `plans/README.md` и фактическое состояние каталога больше не противоречат
  друг другу.
- `git diff --check` проходит.

## 6. Риски

- Самый опасный риск — заархивировать bundle, который ещё нужен как active
  workstream.
- Поэтому переносить можно только bundles с явным сигналом завершения из
  текущей истории работы и/или completed review trail.
